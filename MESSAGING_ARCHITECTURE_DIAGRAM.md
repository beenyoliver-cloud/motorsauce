## Performance Optimization

### Current Setup
```
Polling Interval: 5 seconds
Blocked During Send: 1 second
Average Message Count: < 50 per thread
Database Queries: Indexed on thread_id, user_id
```

### If Performance Issues
```
Option 1: Increase polling to 10s
  setInterval(() => loadData(false), 10000)

Option 2: Add pagination
  const limit = 50;
  const offset = (page - 1) * limit;
  const { data } = await supabase
    .from("messages")
    .select()
    .eq("thread_id", threadId)
    .order("created_at", { ascending: true })
    .range(offset, offset + limit - 1);

Option 3: WebSocket (real-time)
  Replace polling with Supabase Realtime:
  const channel = supabase
    .channel('messages')
    .on('postgres_changes', {
      event: 'INSERT',
      schema: 'public',
      table: 'messages',
      filter: `thread_id=eq.${threadId}`
    }, (payload) => {
      setMessages(prev => [...prev, payload.new]);
    })
    .subscribe();
```
